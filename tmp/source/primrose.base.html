_generate_queries()¶

generate final queries, using parameter substitution, if requested

Note

query_json should be structured as: “query_json”: [ { “query”: “somequery.sql”, “parameters”: { “x”: 1, “y”: 2 } }, { “query”: “someotherquery.sql”, “parameters”: { “x”: 3, “y”: “somestring” } } ] where parameters is an optional key.

This generator will return each query string with variables substituted from parameters (if any)

static _substitute_query(query_json)¶

Substitue our paramters in a query string. Renders the file as a jinja template to allow for advanced query nesting and logic. Uses a jinja environment which searches for templates (sql queries) using relative path and absolute path.

Note

given JSON:

{

“query”: “somequery.sql”, “parameters”: {

“x”: 3, “y”: “somestring”

}

}

and somequery.sql: ` SELECT * from table_01 where x = {x} and y = {y} `

This function will read in the SQL file via jinja, and if parameters is present, substitue out the parameters, returning the final query string

Returns

JSON

Return type

query_json (JSON)
